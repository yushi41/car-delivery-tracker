# 納車待ち管理アプリ - 要件定義書

## 1. プロジェクト概要
**アプリ名**: 納車待ち管理アプリ  
**目的**: 中古車の納車待ち状況をシンプルに管理する  
**方針**: 最低限の機能から始め、必要に応じて拡張する

---

## 2. 基本要件（検討中）

### 2.1 利用者
- [x] **利用者の人数**: 初期は1人（自分のみ）、将来的に3人程度の社内運用を想定
- [x] **利用デバイス**: iPhone（メイン）+ Windows PC（サブ）※両方対応必須
- [x] **アクセス方法**: ブラウザベースのウェブアプリ（どのデバイスからもアクセス可能）
- [ ] インターネット接続の有無（オンライン必須？オフラインでも使いたい？）

### 2.2 管理対象
- [x] **管理する車の台数**: 複数台を同時に管理
- [x] **運用形態**: 複数の車両をまとめて一覧管理
- [x] **担当者管理**: 各車両に営業担当者を割り当てて判別可能にする

---

## 3. 必要最低限の機能（検討中）

### 候補機能リスト
以下から必要なものを選択・追加していきます：

#### 基本情報の管理
- [ ] 車種・メーカー
- [ ] 購入日
- [ ] 購入価格
- [ ] 購入店舗名
- [ ] 担当者名・連絡先

#### 日程管理
- [ ] 納車予定日
- [ ] 納車までの残り日数表示（カウントダウン）
- [ ] 購入からの経過日数

#### 進捗管理
- [ ] 手続きチェックリスト（例：契約完了、支払い完了、車庫証明取得、車検証取得など）
- [ ] 現在のステータス（契約済み、書類手続き中、整備中、納車準備完了など）

#### その他
- [ ] メモ欄（自由記入）
- [ ] 写真の保存
- [ ] 連絡履歴の記録

## 4. 最低限必要な機能（確定）

### 4.1 コア機能（優先度：高）

#### 📅 納車予定日の表示機能
- **カレンダー形式**: 月単位でカレンダー表示、納車予定日に車両情報を表示
- **カウントダウン形式**: 各車両ごとに「あと○日」と表示
- **リスト形式**: 納車予定日が近い順に一覧表示（検討中）

#### 🚗 車両情報管理
- 車種・メーカー
- 購入日
- 納車予定日
- 担当営業者名
- ステータス（例：契約済み、整備中、納車準備完了など）

#### 👤 担当者管理
- 各車両に営業担当者を割り当て
- 担当者でフィルタリング・検索可能

### 4.2 あると嬉しい機能（優先度：中）
- 📆 **Googleカレンダー連携**: 納車予定日をGoogleカレンダーに自動追加
- 📝 メモ機能
- 💰 購入価格の記録
- 🏢 販売店情報（店舗名、連絡先）

### 4.3 将来的な拡張候補（優先度：低）
- 📸 車両写真の添付
- 📞 連絡履歴の記録
- 🔔 納車日が近づいたら通知
- 📊 統計情報（平均納車期間など）

## 5. 技術スタック（確定）

### 5.1 デプロイ・ホスティング
- **Vercel** - 高速、簡単デプロイ、自動スケーリング

### 5.2 フロントエンド
- **Next.js 14** (App Router) - Reactベースのフレームワーク、Vercelと最高の相性
- **TypeScript** - 型安全性でバグを減らす
- **Tailwind CSS** - モダンで美しいデザインを素早く実装
- **shadcn/ui** - 高品質なUIコンポーネントライブラリ
- **Framer Motion** - 滑らかなアニメーション効果

### 5.3 データベース・バックエンド
- **Vercel Postgres** または **Supabase** - クラウドデータベース
- **Prisma** - データベース操作を簡単にするORM

### 5.4 カレンダー連携
- **Google Calendar API** - 納車予定日をGoogleカレンダーに同期

### 5.5 認証（将来の社内運用向け）
- **NextAuth.js** または **Clerk** - ユーザー認証

### 5.6 なぜこの技術スタック？
- ✅ **Vercel + Next.js**: 相性抜群、デプロイが超簡単
- ✅ **Tailwind CSS + shadcn/ui**: 美しく洗練されたデザインを高速で実装
- ✅ **TypeScript**: エラーを事前に防ぎ、開発効率UP
- ✅ **Framer Motion**: Appleのような滑らかなアニメーション
- ✅ **全てモダンな技術**: 最新のベストプラクティス

## 6. 画面構成（提案）

### 6.1 メイン画面（ダッシュボード）
以下の表示形式から選択可能：

#### パターンA: カードビュー（推奨）
各車両を個別のカードで表示し、以下の情報を含む：
- 車種・メーカー
- **大きくカウントダウン表示**（あと○日）
- 納車予定日
- 担当者名
- ステータスバッジ（色分け）
- カードを納車予定日が近い順に並べる

#### パターンB: テーブルビュー
表形式で一覧表示：
- 列：車種 | 納車予定日 | カウントダウン | 担当者 | ステータス
- ソート・フィルタリング機能付き

#### パターンC: カレンダービュー
月単位のカレンダー表示：
- 各日付に納車予定の車両を表示
- 日付クリックで詳細表示

**提案**: モバイルでは「カードビュー」、PCでは「テーブルビュー」と「カレンダービュー」をタブで切り替え

### 6.2 その他の画面
- 車両登録・編集画面
- 車両詳細画面
- 設定画面（担当者管理、Googleカレンダー連携設定など）

## 7. デザインコンセプト ✨

### 7.1 目指すもの
**「世界をとるアプリ」として、Appleのような洗練された美しさ**
- ユーザーに「ワクワク感」を提供
- 「すごい！」と感じさせるUI/UX
- 直感的で、使っていて気持ちいい体験

### 7.2 デザイン原則

#### 🎨 ビジュアルデザイン
- **ミニマリズム**: 無駄を削ぎ落とした、洗練されたデザイン
- **余白の美**: 十分な余白（ホワイトスペース）で視覚的な呼吸感
- **タイポグラフィ**: 読みやすく美しいフォント選定
- **カラーパレット**: 
  - プライマリ: 深みのあるブルー系（信頼感、プロフェッショナル）
  - アクセント: 明るいグリーン系（納車＝希望・新しい始まり）
  - ダーク/ライトモード両対応

#### ✨ アニメーション・インタラクション
- **マイクロインタラクション**: ボタン押下、ホバー時の滑らかな反応
- **ページ遷移**: フェードイン・スライドなど自然な動き
- **カウントダウン**: 数字が変わる際のアニメーション効果
- **カードのホバー効果**: 浮き上がる、影が変わるなど

#### 📱 レスポンシブ・モバイルファースト
- **iPhone最適化**: タップしやすいボタンサイズ（最低44x44px）
- **スムーズスクロール**: 慣性スクロール、スナップ効果
- **ジェスチャー対応**: スワイプで削除、引っ張って更新など

#### 🎯 UX（ユーザー体験）
- **ローディング状態**: スケルトンスクリーン（読み込み中も美しく）
- **エラーハンドリング**: フレンドリーなエラーメッセージ
- **空状態**: 初回訪問時の美しいウェルカム画面
- **フィードバック**: 操作の結果を即座に視覚的に伝える

### 7.3 参考となるデザイン
